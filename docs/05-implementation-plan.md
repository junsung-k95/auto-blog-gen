# 5단계. 구현 계획 (Implementation Plan)

> 4단계까지 정의된 39개 ⭐ MVP 기능을 어떤 순서로 출시할지, 무엇이 무엇을 막는지, 어떤 리스크가 있는지 정의.

---

## 5.1 마일스톤 전략

> **원칙**: 매 마일스톤은 *자체로 출시 가능한 가치 단위*. 이전 마일스톤만으로도 사용자에게 의미가 있어야 함.

| 마일스톤 | 코드명 | 기간 | 핵심 가치 | 출시 가능 형태 |
|---|---|---|---|---|
| **M0** | *현재 상태* | (완) | 음성·사진으로 글 작성 + 즉시 발행 | 이미 동작 중 |
| **M1** | Foundation | 1주 | 새 IA·디자인 + 기존 기능 재배치 | 보기 좋고 확장 가능한 셸 |
| **M2** | Data & Discovery | 2주 | DB·인증 + 황금 키워드 발굴 + 작성 컨텍스트 | "뭘 쓸지" 자동 추천 |
| **M3** | Monetization Hooks | 2주 | 쿠팡파트너스 연동 + SEO 점수판 | 글에 자동으로 돈 붙기 |
| **M4** | Schedule & Pipeline | 1주 | 예약 발행 + 캘린더 + 검수 대기함 | 꾸준한 발행 자동화 |
| **M5** | Performance & Revenue | 2주 | 통계 동기화 + 글별 수익 매칭 + 대시보드 | 깜깜이 탈출 |
| **M6** | Sponsor & Polish | 1주 | 협찬 칸반, 알림 센터, 모바일 마무리 | 종합 수익 도구 완성 |

**총 9주** 가정 (1인 풀타임 기준). 인력·외부 의존성에 따라 ±30%.

## 5.2 마일스톤별 범위 (Scope)

### 🏗️ M1 — Foundation (1주)

> **목표**: "구려 보이는 단일 페이지"를 "확장 가능한 SaaS 셸"로.

**포함 기능**
- X.6 모바일 우선 반응형 (디자인 토큰)
- X.7 다크모드 토큰 정의
- 4.2 IA: 사이드바 9개 메뉴 + 모바일 하단 탭
- 4.3 화면 셸: 대시보드(스텁), 작성(기존 기능 3-패널화), 설정(탭)
- 기존 기능 (음성/사진/생성/발행/트렌드/히스토리) 새 화면에 재배치

**비포함 (다음 단계로)**
- DB·인증 (M2)
- 키워드 발굴 실데이터 (M2)
- 쿠팡 연동 (M3)
- 예약 발행 (M4)

**완료 기준 (DoD)**
- [ ] 모든 라우트 (`#/dashboard`, `#/write`, `#/settings`, ...) 접근 가능
- [ ] 데스크탑·모바일 양쪽 정상 렌더 (Chrome DevTools 검증)
- [ ] 기존 생성·발행 플로우가 그대로 동작
- [ ] 라이트하우스 Accessibility ≥ 90

---

### 🗃️ M2 — Data & Discovery (2주)

> **목표**: 사용자가 *"뭘 쓸지"* 더 이상 고민 안 함.

**포함 기능**
- X.1 사용자 인증 (이메일+패스워드, JWT)
- X.2 데이터 영속화 (SQLite 시작 → 추후 Postgres)
- 0.1 네이버 블로그 다중 프로필 등록 (DB 저장)
- 0.3 과거 글 자동 임포트
- 0.5 운영 카테고리·주력 키워드
- 0.9 API 키 안전 저장 (AES-256-GCM)
- 1.1 황금 키워드 추천 엔진 (스코어링 v1)
- 1.2 키워드 데이터 수집 (블로그·카페 검색량 + 데이터랩)
- 1.3 경쟁도 지표
- 1.5 시즌 트렌드
- 1.8 키워드 북마크·블랙리스트
- 1.9 카테고리별 추천
- X.3 백그라운드 잡 큐 (refresh_keywords 일 1회)
- 2.1 키워드 → 작성 컨텍스트 주입

**완료 기준**
- [ ] 신규 사용자가 가입 → 블로그 연결 → 1분 안에 키워드 카드 리스트 받음
- [ ] `keyword_snapshots` 일별 적재 동작
- [ ] 추천 API 응답 ≤ 1s (캐시 적중 시)

---

### 💰 M3 — Monetization Hooks (2주)

> **목표**: 글을 쓰는 순간 돈이 자동으로 붙음.

**포함 기능**
- 0.6 수익 채널 연결 (쿠팡 파트너 ID 등록)
- 2.5 SEO 구조 자동 생성 (키워드 기반 H2/H3 골격)
- 2.8 쿠팡 상품 자동 매칭
- 2.9 제휴 링크 자동 삽입 + 의무 고지
- 2.10 협찬/내돈내산 표기 토글
- 2.12 글 길이 프리셋
- 3.1 SEO 점수판 (9개 항목 100점 만점)
- 3.2 저품질 위험 진단
- 3.4·3.5 추천 제목·태그 (현 트렌드 분석 흡수)
- 3.6 메타정보 검증

**완료 기준**
- [ ] 본문에서 상품 매칭률 ≥ 70% (수동 테스트 케이스 20건)
- [ ] 발행 전 SEO 점수 항상 표시
- [ ] 협찬글은 공시 문구 누락 시 발행 차단

## 5.3 의존성 그래프

```
                        ┌─────────────────────┐
                        │  M1  Foundation     │
                        │  IA + 디자인 + 셸    │
                        └──────────┬──────────┘
                                   │
                                   ▼
                        ┌─────────────────────┐
                        │  M2  Data &         │
                        │      Discovery       │
                        │  (인증·DB·키워드)    │
                        └──────────┬──────────┘
                                   │
                ┌──────────────────┼──────────────────┐
                ▼                  ▼                  ▼
   ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
   │ M3 Monetization │  │ M4 Schedule &   │  │  (병렬 가능)     │
   │     Hooks        │  │    Pipeline      │  │                  │
   └────────┬────────┘  └────────┬────────┘  └─────────────────┘
            │                    │
            └──────────┬─────────┘
                       ▼
            ┌─────────────────────┐
            │  M5 Performance &   │
            │     Revenue          │
            └──────────┬──────────┘
                       │
                       ▼
            ┌─────────────────────┐
            │  M6 Sponsor &       │
            │     Polish           │
            └─────────────────────┘
```

**핵심 의존**
- M2 → M3·M4: DB·인증 없으면 글 단위 영속화 불가
- M3·M4 → M5: 쿠팡 링크·발행 이력이 있어야 수익 매칭 가능
- M5 → M6: 성과 데이터가 있어야 협찬 가격 산정·리포트 의미 있음

## 5.6 리스크·완화책

| 리스크 | 영향 | 가능성 | 완화 |
|---|---|---|---|
| 네이버 OpenAPI 일일 호출 한도 (25k/일) | M2 키워드 추천 막힘 | 중 | 키워드 사전 계산 + 캐시 (24h). 사용자별 호출량 한도 도입 |
| 네이버 블로그 통계 페이지 구조 변경 | M5 통계 스크래핑 깨짐 | 높음 | 셀렉터를 설정값으로 추출, 변경 감지 시 알림 |
| 쿠팡파트너스 API 승인 (월 거래액 조건) | M3 상품 매칭 막힘 | 중 | 미승인 시 검색 URL만 노출하는 폴백 (수익은 못 잡지만 사용은 가능) |
| AI 토큰 비용 폭주 | 운영비 적자 | 중 | 사용자별 월 비용 한도 + 모델 자동 다운그레이드 옵션 |
| 네이버 XML-RPC 발행 실패율 | M4 예약 발행 신뢰도 | 중 | 3회 재시도 + 토큰 만료 자동 감지 |
| 저품질 블로그 판정 | 사용자 신뢰 손상 | 높음 | M3에 저품질 위험 진단 강제, 발행 차단 옵션 |
