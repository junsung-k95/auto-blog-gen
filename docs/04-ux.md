# 4단계. UX/UI 설계

> 3단계 데이터 모델·API를 어떤 화면과 인터랙션으로 사용자에게 노출할지 정의.
> 모바일(이동 중 음성녹음) 우선, 데스크탑은 풍부한 데이터 분석 뷰.

---

## 4.1 디자인 원칙

| 원칙 | 의미 |
|---|---|
| **수익이 보이는 UI** | 모든 화면 최상단에 오늘/이번 달 수익 위젯. 글 카드마다 누적 수익 뱃지. |
| **데이터로 결정** | 추측 대신 숫자. 키워드 카드·글 카드·상품 카드 모두 핵심 지표 3개 이내로 한눈에. |
| **한 번에 한 액션** | 화면당 주요 CTA 1개. 보조 액션은 시각적으로 약하게. |
| **모바일 = 입력, 데스크탑 = 분석** | 모바일은 음성녹음·발행 승인에 최적화, 데스크탑은 대시보드·캘린더 중심. |
| **소음 차단** | 알림 우선순위(긴급/중요/일반) 분리, 부진 글 알림은 일일 1회 묶음. |

## 4.2 정보 구조 (IA)

```
┌─ 좌측 글로벌 내비게이션 ─┐  ┌───── 콘텐츠 영역 ─────┐
│                       │  │                      │
│  🏠 대시보드            │  │                      │
│  🔍 키워드 발굴          │  │   현재 선택된 메뉴의   │
│  ✍️  작성              │  │   콘텐츠 표시         │
│  📋 검수 대기함          │  │                      │
│  📅 발행 캘린더          │  │                      │
│  📊 성과               │  │                      │
│  💰 수익 관리           │  │                      │
│  🤝 협찬 보드           │  │                      │

```

### 메뉴 ↔ 기능 매핑

| 메뉴 | 주요 화면 | 매핑 기능 |
|---|---|---|
| 🏠 대시보드 | 홈 | Phase 1 추천 미리보기 + Phase 5 핵심 지표 + Phase 4 오늘 발행 |
| 🔍 키워드 발굴 | 카드 리스트 + 상세 | 1.1~1.11 |
| ✍️ 작성 | 3-패널 에디터 | 2.1~2.16, 3.1~3.8 |
| 📋 검수 대기함 | 칸반 (draft/review/scheduled) | 4.5, 4.6 |
| 📅 발행 캘린더 | 월/주 뷰 | 4.2~4.4 |
| 📊 성과 | 차트 대시보드 + 글별 카드 | 5.1~5.9 |
| 💰 수익 관리 | 채널 통합 + 제휴 링크 | 6.1~6.3, 6.6~6.7 |
| 🤝 협찬 보드 | 칸반 | 6.4~6.5 |
| ⚙️ 설정 | 탭 (블로그/채널/AI/알림/사용량) | 0.1~0.9, X.5 |

### 모바일 IA (≤768px)

좌측 사이드바 → **하단 탭 + 햄버거 메뉴** 로 변환:

```
하단 탭 (4개): 🏠 홈  ✍️ 작성  📅 캘린더  💰 수익
나머지 (발굴/검수/성과/협찬/설정): 햄버거 ☰
```

## 4.4 인터랙션 패턴

### 4.4.1 글로벌 단축키 (데스크탑)

| 단축키 | 동작 |
|---|---|
| `⌘ + K` | 명령 팔레트 (검색·이동) |
| `⌘ + N` | 새 글 작성 |
| `⌘ + Enter` | 미리보기에서 발행 다이얼로그 |
| `⌘ + S` | 임시저장 (자동저장 외 명시) |
| `Esc` | 슬라이드 패널 닫기 |
| `J / K` | 키워드/글 카드 리스트 위·아래 이동 |
| `?` | 단축키 가이드 토글 |

### 4.4.2 FAB (Floating Action Button)

화면 우측 하단 고정. 컨텍스트별 동작:

| 화면 | FAB |
|---|---|
| 대시보드/검수함 | ⊕ 새 글 작성 |
| 캘린더 | ⊕ 일정 추가 |
| 발굴 | ☆ 북마크한 키워드 보기 |
| 모바일 모든 화면 | 🎤 빠른 음성 메모 (어디서든 1탭 녹음) |

### 4.4.3 슬라이드 패널 (Drawer)

다음 경우 **우측에서 슬라이드 인** (현재 히스토리 패널과 동일 패턴 확장):

- 키워드 카드 클릭 → 키워드 상세
- 글 카드 클릭 → 글 성과 카드
- 알림 벨 클릭 → 알림 센터
- 협찬 카드 클릭 → 협찬 상세·진행 노트

### 4.4.4 호버링/마이크로 인터랙션

| 위치 | 호버 시 |
|---|---|
| 본문 문단 | "이 문단 다시 쓰기" 미니 버튼 |
| 본문 이미지 | "캡션 생성" / "alt 텍스트 편집" |
| 키워드 카드 | "이걸로 쓰기" + 북마크 ☆ |
| 글 성과 행 | "리라이트 제안" / "공유" |
| 쿠팡 상품 카드 | "본문 삽입" / "즐겨찾기" |

### 4.4.5 빈 상태 (Empty State)

| 화면 | 빈 상태 메시지 |
|---|---|
| 대시보드(첫 방문) | "블로그를 연결하면 오늘의 추천이 시작됩니다 → [블로그 연결]" |
| 키워드 발굴(데이터 적재 전) | "키워드 분석 중... 보통 5분 이내 완료" + 진행 바 |
| 검수 대기함 | "초안이 없어요. 음성으로 빠르게 시작해볼까요? 🎤" |
| 성과 | "발행한 글이 모이면 여기에 성과가 표시됩니다" |
| 수익 관리 | "수익 채널을 연결하면 채널별 수익이 자동 집계돼요" |

### 4.4.6 알림 (Notification Center)

우측 슬라이드 패널. 3단계 우선순위:

| 우선순위 | 예 | 표시 방식 |
|---|---|---|
| 🚨 긴급 | 예약 발행 실패 / 자격증명 만료 | 토스트 + 벨 빨강 |
| ⭐ 중요 | 신규 협찬 제안 / 부진 글 발견 | 벨 뱃지 |
| ℹ️ 일반 | 발행 완료 / 일일 추천 | 묶음 다이제스트 |
